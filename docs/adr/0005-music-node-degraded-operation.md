# Music node degraded operation

Botato must stay logged into Discord and run non-music feature modules when the music node is unreachable. Music is a degradable capability, not a process dependency: `MUSIC_NODE_*` stays required at boot (the workload is still part of the product), but a down or late-starting node means degraded music with indefinite Shoukaku reconnect (`reconnectTries` beyond the library default of 3), not a dead bot. Operators get clear logs on disconnect/reconnect; guild members get a clear reply when they use a music command while the node is down, and Discord presence shows a warning (e.g. “Music unavailable”) only while degraded — cleared when healthy, with no channel announcements on loss or recovery. If the music node is lost mid-play, the music session ends (queue and voice are dropped); surviving or auto-resuming sessions across node death is out of scope for v1.
